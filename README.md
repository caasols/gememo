# Gememo

[![CI](https://github.com/caasols/gememo/actions/workflows/ci.yml/badge.svg)](https://github.com/caasols/gememo/actions/workflows/ci.yml)

> Automated, bot-free Google Meet notes — saved to Craft, Apple Notes, or Obsidian the moment you leave a call.

When you click the red **Leave call** button, this Chrome extension intercepts it, asks Google's own Gemini AI to summarise what happened, and saves a formatted note to your note-taking app — silently, in the background, without stealing focus.

**No API key. No subscription. No bot joining your call. macOS only.**

---

## Why this vs. paid tools

Most AI meeting note tools (Fireflies, Otter.ai, Granola, tl;dv, Fathom) work by joining your meeting as a **bot participant** — a visible third party that records audio, ships it to their cloud, and charges a monthly subscription. This extension works fundamentally differently:

| | This extension | Paid bots (Fireflies, Otter, etc.) |
|---|---|---|
| **How it captures** | Reads Gemini's existing transcript in the Google Meet DOM | Joins as a bot, records audio |
| **Data leaves your machine** | Never — notes generated by Gemini, processed locally | Audio/transcript sent to vendor cloud |
| **Bot visible to participants** | No | Yes — "Fireflies Notetaker" joins your call |
| **Requires consent / disclosure** | No | Varies by jurisdiction |
| **Cost** | Free | $10–20+/user/month |
| **Works without vendor uptime** | Yes | No |

> **Core advantage:** This extension uses Google's own Gemini AI that is already running inside your meeting. There is no subscription, no bot, and no audio leaving your machine. The quality comes from Gemini — the same model paid tools approximate with their own transcription pipelines.

## Prerequisites

Before installing, make sure you have:

- **macOS** (native host is macOS-only)
- **Google Chrome** (MV3 extension)
- **Google Workspace account** with Gemini in Google Meet enabled ([check here](https://workspace.google.com/intl/en/products/gemini/))
- **A supported output app** — Craft ([craft.do](https://craft.do)), Apple Notes, or Obsidian (all included)
- **Python 3.9+** (`python3 --version`)

## Installation

### Step 1 — Load the Chrome extension

1. Open **`chrome://extensions`** in Chrome
2. Enable **Developer mode** (toggle in the top-right corner)
3. Click **Load unpacked** and select the **`extension/`** folder from this repo
4. Note the **32-character extension ID** shown below the extension name — you'll need it in step 2

### Step 2 — Install the native messaging host

The native host is a Python script that receives the transcript from the extension and pushes it to Craft.

```bash
cd native_host
bash install.sh
```

The script will ask for the extension ID from step 1, then write the Chrome native messaging manifest to:
```
~/Library/Application Support/Google/Chrome/NativeMessagingHosts/
```

> **Verify the host is connected:** Open the extension popup — the Main tab shows a green dot with "Native host ready".

### Step 3 — Configure your output destination

Open the extension popup → **Settings tab** → **Output app** → choose one:

| App | Notes |
|---|---|
| **Craft** | Default. Optionally set an inbox folder ID (right-click folder → Copy Deeplink → use the `docId`). |
| **Apple Notes** | Zero config. Notes land in the default Notes folder. |
| **Obsidian** | Click the Vault folder field to pick your vault. Gememo writes YAML-frontmatted `.md` files that Obsidian picks up automatically. |

### Step 4 (Optional) — Configure file backup

In **Settings → File backup**, enable the toggle and set a local folder. Every snapshot and final note will also be saved as a `.md` file there — useful for recovery if Craft fails.

## How it works

```
Google Meet — in-meeting (periodic snapshot every 8 min, configurable 3–30 min)
  └─ Chrome extension (content_meet.js)
       ├─ auto-activates Gemini via CDP trusted events (isTrusted=true)
       ├─ injects summary prompt, waits for Gemini to finish streaming
       ├─ caches result in memory (nothing sent to your note app yet)
       └─ (if file backup enabled) writes timestamped snapshot to backup folder

Google Meet — Gemini deactivates mid-meeting (e.g. other person leaves a 1:1)
  └─ captureProactively() fires immediately
       ├─ uses cached snapshot if available
       └─ attempts live Gemini capture if no snapshot yet (60 s timeout)

Google Meet — red Leave call button clicked
  └─ Chrome extension intercepts the click
       ├─ mutes cam and mic
       ├─ waits for any running snapshot to complete naturally
       ├─ runs a fresh Gemini capture (60 s timeout), falls back to cache on failure
       └─ sends transcript to native messaging host
            └─ native_host/meeting_minutes_host.py
                 ├─ formats markdown (strips bold/backtick markers)
                 ├─ writes final backup file (if enabled)
                 ├─ routes to output app (Craft / Apple Notes / Obsidian)
                 │    Craft: opens craftdocs://createdocument?content=... (no file access needed)
                 │    Apple Notes: osascript with HTML body
                 │    Obsidian: writes YAML-frontmatted .md to vault folder
                 ├─ on failure: stores mm2c_last_failed; retry widget appears in popup
                 └─ macOS notification on success
```

## Daily use

Join Google Meet normally. The extension auto-activates the Gemini panel at meeting start. When you're done, click the red **Leave call** button — that's it.

The extension will:
1. Mute your cam and mic
2. Run a final Gemini summary of the full meeting
3. Format and save the note to Craft in the background
4. Show a macOS notification when done

**The popup (click the extension icon) has 5 tabs:**

- **Main** — live status: whether a meeting is active, last snapshot time, host connection, retry widget on failure
- **Rules** — default prompt editor + per-meeting-type prompt rules (e.g. standup format for daily scrums)
- **Settings** — snapshot frequency, note language, output destination, file backup
- **Logs** — full activity log grouped by meeting, with per-entry Retry button on errors
- **About** — version, GitHub link, extension ID copy button

**Mid-meeting features:**
- **Periodic snapshots** — every 8 minutes (configurable 3–30 min) the extension captures the current Gemini summary in memory. Snapshots also go to the backup folder if enabled.
- **"Last snapshot: N min ago ▸"** — in the Main tab, shows when the last snapshot was taken with an expandable preview.
- **"Capture now" button** — visible at the bottom of the popup when in a meeting. Triggers a snapshot immediately.

**Recovery:** If Craft fails, the extension retries once with the most recent snapshot file. On persistent failure, the popup shows the exact path to the backup file on your Mac.

## Gemini prompt

The default prompt is stored in `extension/constants.js` (`DEFAULT_PROMPT`) and can be customised from the popup **Rules tab** without touching code. The prompt asks Gemini to produce:

- **Attendees** — list of participants by name
- **Summary** — concise purpose and outcome
- **Key Points** — detailed bullets with topic labels
- **Decisions Made** — concrete agreements reached (only where explicitly agreed)
- **Action Items** — owner + deadline (no ambiguous pronouns)
- **Open Questions** — unresolved questions and concerns raised
- **Next Steps** — shared calendar commitments and scheduled follow-ups *(coming soon)*

You can also set **per-meeting rules** in the Rules tab: match a meeting title with a regex to use a completely different prompt. Example: a `standup` regex routes daily standups to a Blockers/Done/Next format.

**Reset to default** at any time via the "Reset to default" button in the Rules tab.

## Troubleshooting

| Symptom | Fix |
|---|---|
| Leave button behaves normally (no capture) | Extension didn't attach. Check it's enabled in `chrome://extensions` and reload the Meet tab. |
| Red `!` badge on extension icon | Click the icon — the Main tab shows the error. Usually the native host isn't installed. |
| "Native host not found" in popup | Run `bash native_host/install.sh`. Make sure you use the correct extension ID. |
| Popup shows "open the Gemini panel to enable capture" | Gemini wasn't activated. The extension will try to activate it automatically — if it shows this message, hover over the ✦ icon in the Meet toolbar and click "Start now". |
| "Gemini may be disabled for your account" | Your Google Workspace admin may have blocked Gemini. Check with your admin. |
| No Craft note appears after leaving | Check the **Logs tab** — if it says "Saved to Craft", the note was created. If not, the popup will show the path to the local backup file. |
| Craft note appeared but is empty | Gemini was active but produced no transcript (very short meeting or silent call). The backup file at the shown path has the raw content. |
| Selectors broken after a Meet update | Run `npm test` (Playwright) or paste `tests.js` into the Meet DevTools console and run `MM2C_TESTS.run()` to identify which selectors fail. |
| Host version mismatch warning | Re-run `bash native_host/install.sh` to update the native host. |
| Two identical Craft notes created | Rare dedup race condition — already tracked as U4 in the roadmap. The second note is safe to delete. |

## DOM selectors (verified 2026-05-31)

| Element | Selector / Notes |
|---|---|
| Leave call button | `button[aria-label="Leave call"]` |
| Call controls bar | `div[aria-label="Call controls"]` |
| Gemini toggle — not started | `[role="button"]` text `"Geminispark_off"` — CDP hover required to reveal "Start now" tray |
| Gemini toggle — active | `button[aria-label="Gemini"]` — CDP click required on first open |
| "Start now" button | `button` text `/start now/i` — appears in hover tray; CDP `isTrusted` click required |
| Gemini input | `div[aria-label="Ask Gemini"][contenteditable="true"]` |
| Submit button | `button[aria-label="Submit"]` |
| Side panel | `aside[aria-label="Side panel"]` |

**Why CDP?** Meet's "Start now" button and the Gemini panel toggle both require `isTrusted=true` events. The extension uses `chrome.debugger` + CDP `Input.dispatchMouseEvent` to simulate trusted mouse events. Chrome shows a brief "DevTools in use" infobar during activation (~1 second).

Google occasionally updates Meet's DOM. Run `npm test` to verify selectors against the current DOM.

## File structure

```
gememo/
├── extension/
│   ├── manifest.json          — Chrome MV3 manifest ("debugger" permission required)
│   ├── constants.js           — DEFAULT_PROMPT and shared helpers
│   ├── content_meet.js        — Leave intercept, Gemini flow, snapshots, CDP automation
│   ├── background.js          — service worker: native host bridge, logs, dedup, CDP relay
│   ├── popup.html / popup.js  — 4-tab popup: Main, Rules, Settings, Logs
│   └── tests.js               — injectable test suite (also runs via Playwright)
├── tests/
│   ├── fixture.html           — HTML fixture for headless Playwright tests
│   └── run_tests.spec.js      — Playwright spec: runs MM2C_TESTS.run() headlessly
├── native_host/
│   ├── meeting_minutes_host.py — receives transcript, formats, pushes to Craft with retry
│   ├── install.sh             — one-time setup: registers host with Chrome
│   ├── io.gememo.host.json              — native messaging manifest template
│   ├── test_snapshot.py       — snapshot write + pruning tests
│   ├── test_craft_push.py     — x-callback-url success/failure/timeout tests
│   └── test_parse_transcript.py — markdown stripping + heading detection tests
├── scripts/
│   └── push_to_craft.py       — x-callback-url push with local HTTP callback listener
├── package.json               — npm scripts (npm test runs Playwright suite)
└── playwright.config.js       — Playwright config
```

## Contributing

Contributions welcome. The [ROADMAP.md](ROADMAP.md) has a prioritised work queue — Phase 1 items (prompt text improvements) are the easiest starting point and have the highest impact per line changed.

**Before submitting a PR:**
- Run `npm run test:all` — all tests should pass (CI runs the same command)
- DOM selector changes should be verified against a live Google Meet session

**To report a broken selector:** Google updates Meet's DOM periodically. Open an issue with the selector name, the old value, and the new value you found in DevTools. Fixes are usually one-line.
